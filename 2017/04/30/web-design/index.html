学了html与css基础后

发表于 2017-04-30   |   分类于 前端

前端学习的开始是比较简单的。只需要记住一些标签和属性，用语言来形容一个带样式的文本

html的难点在于语义和其设计哲学。

html形容文本意义，不能决定网页样式。一段纯html文件在浏览器中，被不同标签包围的文本有不同外貌，是因为有浏览器的默认css文件渲染（这是为了普通用户的特殊情况而考虑的）。
写html标签不能根据你想要的样式写，要根据语义。

div标签说的有的有很多，它与css，js的配合使用才能发挥最大用处。所以div和css，div和js一起谈才是最好的。单拿出来讲，是意义不大的。

css有什么难的呢？

我一开始学习css的时候，最大的问题是“原来还有这样的属性”，后来发现这是最简单的。
涉及到浏览器，设备方面的问题，书上大都是没有的

  • 继承，它既会产生你意想不到的效果，也会让你少写一写代码。
  • 选择器，css的选择器有许多种，让你准确定位到需要的标签。选择是一方面，为标签写class时，可以写多个class，一个是本身特殊的，其他的可以是通用的，这样能写的更精简。
  • 盒子模型，div的布局，div是一个块元素，是一种容器（盒子），它的用法一定要理解盒子模型（标准），div区块所包裹的内容的位置会在哪里，最好是写之前就知道在哪里。（然而有时侯真是需要边调边写）

不论是编程还是写网页，精简与优化是必不可少，有简单的实现方法，就不要用笨的。服务器的性能是有限的，网络带宽是有限的。代码的精简能大大提升效率和用户体验。维护也更方便。

CSS 更多是学习设计，css只是工具，如何画好，如何部好局才是重点。（布局难）

现在人人都有移动端设备，网页的设计也变得更加麻烦，适配不同的屏幕（尺寸，高分屏）。现在html5和css3的广泛支持，浏览器标准的统一（？），也算简单下了。我想还用IE9以下的只用公家单位和学校机房了。
